Velocity Check Information

Velocity checks monitor the frequency of transactions within a specific timeframe. If the number of transactions exceeds a predefined threshold, the system flags the activity as potentially fraudulent. This is useful for detecting suspicious rapid transaction patterns.

How It Works: The system counts the number of transactions within the past hour and compares it against a threshold (e.g., 10 transactions). If the count exceeds the threshold, the transaction is flagged.

Location Check Information

Location checks verify the consistency of a transaction's location compared to previous transactions. If the location of a new transaction differs significantly from the previous location, it may be flagged as potentially fraudulent.

How It Works: The system calculates the distance between the current transaction location and the previous transaction location. If the distance exceeds a threshold (e.g., 1000 km), the transaction is flagged.
